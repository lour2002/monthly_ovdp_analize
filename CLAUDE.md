# ОВДП Analysis Routine — Інструкція

## Призначення
Ця інструкція визначає поведінку Claude Code рутини, що запускається скриптом
`analyze_ovdp.py`. Рутина отримує JSON з активними ОВДП, збагачує дані,
та оновлює список на сторінці Notion.

---

## Вхідні дані

Рутина отримує JSON-рядок у полі `text`:

```json
{
  "analysisDate": "YYYY-MM-DD",
  "windowDays": 30,
  "bonds": [
    {
      "isin": "UA4000XXXXXX",
      "couponRate": 16.2,
      "availableQuantity": 16638072,
      "nextCoupon": {
        "date": "YYYY-MM-DD",
        "amount": 81.0
      }
    }
  ]
}
```

Поля:
- `analysisDate` — дата запуску скрипта
- `windowDays` — кількість днів для позначки 🔥 (зазвичай 30)
- `bonds[]` — масив активних ОВДП з inzhur.reit
  - `isin` — код ISIN
  - `couponRate` — річна ставка купону з НБУ (`auk_proc`), %
  - `availableQuantity` — доступна кількість облігацій
  - `nextCoupon.date` — дата наступної купонної виплати
  - `nextCoupon.amount` — сума виплати на одну облігацію (грн), або `null`

## Кроки виконання

### 1. Парсинг і валідація
- Розпарсити JSON із поля `text`.
- Якщо `bonds` порожній — завершити з повідомленням «Активних ОВДП не знайдено».

### 2. Збагачення: ціни купівлі / продажу
- Перейти на сторінку **https://www.inzhur.reit/offer/ovdp**
- Для кожного ISIN зі списку `bonds` знайти на сторінці значення:
  - **Купівля** — ціна купівлі (%)
  - **Продаж** — ціна продажу (%)
  - **Спред** — обчислити як `Продаж − Купівля` (якщо обидва значення наявні)
- Якщо сторінка недоступна або дані для конкретного ISIN не знайдено — залишити поля порожніми та продовжити виконання.

### 3. Читання поточного стану Notion
- Відкрити сторінку **https://www.notion.so/Inzhur-3412ef91da48811685abce048d0cd5ef**
- Зчитати поточний список ISIN у таблиці.
- Зберегти його для порівняння (крок 5).

### 4. Оновлення таблиці ОВДП у Notion
Оновити таблицю на сторінці **https://www.notion.so/Inzhur-3412ef91da48811685abce048d0cd5ef**

Структура таблиці:

| ISIN | Купон | Дата виплати | Купівля | Продаж | Спред | Доступно |
|------|-------|-------------|---------|--------|-------|----------|

Правила заповнення кожного рядка:
- **ISIN** — код із поля `isin`
- **Купон** — значення `couponRate`, формат `16.2%`; якщо відсутнє — «—»
- **Дата виплати** — `nextCoupon.date`; якщо `nextCoupon` відсутній — «—»
  - Якщо дата потрапляє в проміжок `[analysisDate, analysisDate + windowDays]` — додати 🔥 перед датою: `🔥 2026-05-10`
- **Купівля** — ціна купівлі з inzhur.reit; якщо не отримано — порожньо
- **Продаж** — ціна продажу з inzhur.reit; якщо не отримано — порожньо
- **Спред** — `Продаж − Купівля`; якщо одне зі значень відсутнє — порожньо
- **Доступно** — значення `availableQuantity`; якщо `null` — «—»

Алгоритм оновлення:
1. Видалити з таблиці рядки, яких немає у новому списку `bonds`.
2. Оновити існуючі рядки актуальними даними.
3. Додати нові рядки для ISIN, яких ще не було в таблиці.

### 5. Коментар до сторінки
Додати коментар до сторінки Notion з підсумком змін:

```
Оновлено: {analysisDate}

Додано ({N}): UA4000XXXXXX, UA4000YYYYYY, …
Видалено ({N}): UA4000ZZZZZZ, …
Без змін: {N} позицій
```

Якщо жодних змін не було — написати «Список ОВДП без змін».

### 6. Підтвердження
Після успішного оновлення вивести:
```
Сторінку оновлено: https://www.notion.so/Inzhur-3412ef91da48811685abce048d0cd5ef
```

## Обробка помилок
- Якщо Notion недоступний — вивести таблицю у консоль і завершити з кодом 1.
- Якщо `couponRate` відсутній — вказати «—» у колонці «Купон».
- Якщо `nextCoupon` є `null` — вказати «—» у колонці «Дата виплати», без 🔥.
- Якщо `availableQuantity` є `null` — вказати «—» у колонці «Доступно».
- Якщо сторінка inzhur.reit/offer/ovdp недоступна — пропустити крок 2, продовжити з порожніми цінами.
