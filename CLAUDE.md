# ОВДП Analysis Routine — Інструкція

## Призначення
Ця інструкція визначає поведінку Claude Code рутини, що запускається скриптом
`analyze_ovdp.py`. Рутина отримує JSON із відфільтрованими ОВДП, проводить
аналіз і зберігає звіт у Notion.

---

## Вхідні дані

Рутина отримує JSON-рядок у полі `text`:

```json
{
  "analysisDate": "YYYY-MM-DD",
  "windowDays": 30,
  "bonds": [
    {
      "isin": "UA4000XXXXXX",
      "couponRate": 15.5,
      "availableQuantity": 1000,
      "upcomingCoupons": [
        { "date": "YYYY-MM-DD", "amount": 155.0 }
      ]
    }
  ]
}
```

---

## Кроки виконання

### 1. Парсинг і валідація
- Розпарсити JSON із поля `text`.
- Переконатися, що `bonds` містить від 1 до `TOP_N` записів.
- Якщо `bonds` порожній — завершити з повідомленням «Активних ОВДП з найближчими купонами не знайдено».

### 2. Аналіз
Для кожного ОВДП обчислити та зафіксувати:
- **Купонна ставка** (`couponRate`, %) — основний критерій.
- **Найближча дата виплати** — перший елемент `upcomingCoupons[0].date`.
- **Кількість днів до виплати** — різниця між датою виплати та `analysisDate`.
- **Доступна кількість** (`availableQuantity`).

Порівняти два ОВДП між собою:
- Визначити, який має вищу ставку.
- Врахувати, чий купон виплачується раніше.
- Сформулювати коротку рекомендацію (1–2 речення) українською мовою.

### 3. Збереження у Notion

Зберегти результат у базу знань **Фінанси**:
- Notion page ID: `2802ef91da4881ddb1fdf2de7db3a4d7`
- URL: https://www.notion.so/2802ef91da4881ddb1fdf2de7db3a4d7

Структура нового запису:

**Заголовок сторінки:** `ОВДП Аналіз — {analysisDate}`

**Блок 1 — Таблиця (bonds)**

| ISIN | Купон % | Дата виплати | Днів до виплати | Доступна кількість |
|------|---------|--------------|-----------------|-------------------|
| ...  | ...     | ...          | ...             | ...               |

**Блок 2 — Рекомендація**

Короткий текстовий абзац (paragraph block) з рекомендацією щодо вибору між
двома ОВДП. Якщо купонна ставка однакова — рекомендувати той, у якого
виплата раніше.

**Блок 3 — Метадані**
- Дата аналізу: `analysisDate`
- Вікно пошуку купонів: `windowDays` днів
- Джерело даних: inzhur.reit + bank.gov.ua (NBU)

### 4. Підтвердження
Після успішного запису вивести:
```
Звіт збережено: https://www.notion.so/{page-id}
```

---

## Обробка помилок
- Якщо Notion недоступний — вивести таблицю у консоль і завершити з кодом 1.
- Якщо `couponRate` відсутній для якогось ОВДП — вказати «—» у таблиці та
  виключити з порівняння рейтингу.
- Якщо `amount` купону `null` — вказати «н/д» у таблиці.
